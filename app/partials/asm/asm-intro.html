Application Settings Management (ASM)

App settings provide options for how an application should function. For example, by default, the iOS Safari app uses Google to perform Web searches. With ASM, Safari could be remotely configured to use Bing or Yahoo! instead.

The iOS Settings app approach is great for providing a consistent way of managing settings for a single user, but doesn't meet the needs of a mission critical app. Often there needs to be a way to apply settings to fleets of devices at once, rather than making changes on-site, one at a time. ASM makes the following three things possible for mission critical apps:

  1. A consistent way of handling settings and values.
  2. Centralized settings management. A way to go to a Web page, define a number of settings, and then have those become active for your app across many devices.
  3. An option for changing the app settings from the app itself. Ideal for immediate changes directly on that one device

The ASM module of the MokiManage SDK provides a consistent way for an administrator to control settings for apps across multiple devices from a single, centralized location.

Creating the Settings Schema

The first thing you need to do in implementing the ASM module, is define a settings schema for your app. The settings schema defines what settings values you need to expose, and what the relationships are between them. The settings schema is expressed in a JSON file, which you will include with your app:

SettingsSchema.json

Your SettingsSchema.json file can be used for both Android and iOS. However, the updating of settings and presenting the settings in the UI will have to be done differently according to the platform.
