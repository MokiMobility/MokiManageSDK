AEM provides remote monitoring to keep track of what is going on with a purposed app. Capabilities include the following:

  • Device location
  • Screenshots
  • Memory use
  • IP address: Wi-Fi and cellular network
  • Device name
  • Battery status
  • App uptime
  • Device uptime
  • Processor load
  • App version
  • OS version
  • Storage space used and available
  • Logs
  • Hardware monitoring
  • System monitoring

Information collected is available in the MokiManage Web console. The monitoring information can be used to troubleshoot and keep an app up and running.

The app can push logs and screenshots to MokiManage, and MokiManage can also request them at any time.

Having this information available from a centralized console makes it easier to make sure an app is running properly, and helps to troubleshoot a range of hardware, connectivity, and app problems.

Setting up the AEM section of the SDK is different for iOS and Android

iOS setup requires 4 steps:

  1. Setup Apple Push Notification service (APNs) for your app
  2. Add the MokiManage SDK to your Xcode project
  3. Add some code to your app delegate
  4. Verify that AEM is enabled in your app

Android setup requires 6 steps:

  1. Add the MokiManage SDK to your Eclipse project
  2. Setting up the AdnroidManifest.xml
  3. Setting up MokiManage in your application
  4. MokiManage broadcasts
  5. Silently enrolling your device
  6. Verify that AEM is enabled in your app
